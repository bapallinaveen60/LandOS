## 1. Mathematical Framework for Composite Indices

LND-11 structures environmental, hydrological, and meteorological hazards into four primary composite scores ($0\text{--}100$) and a final letter-based Investment Grade.

```
       [ 50+ Geophysical & Climate Spatial Inputs ]
                           |
                           v Zonal Feature Extraction
                 [ Spatial ML Models ]
                           |
            +--------------+--------------+
            |  Flood, Landslide, Heat,    | Hazard Probabilities ($P_i$)
            |  Drought, Erosion, GW Risk  | & Confidence Intervals
            +--------------+--------------+
                           |
                           v Weighted Index Functions
            +--------------+---------------------------------------+
            |                                                      |
            v                                                      v
  [ Climate Risk Score ]                                 [ Water Security Score ]
       (0-100)                                                  (0-100)
            |                                                      |
            +--------------------------+---------------------------+
                                       |
                                       v
                        [ Investment Grade Score ] (AAA-BB)
                                       ^
            +--------------------------+---------------------------+
            |                                                      |
            v                                                      v
[ Land Development Score ]                              [ ESG Risk Score ]
       (0-100)                                               (0-100)
```

### 1.1 Climate Risk Score (CRS)
The $CRS$ represents the cumulative probabilistic hazard profile of the land parcel. It is computed as a weighted combination of dynamic hazard probabilities:
$$CRS = 100 \times \left( \sum_{i \in H} w_i \cdot P(H_i | X_{\text{parcel}}) \right)$$
Where:
* $H = \{\text{Flood}, \text{Drought}, \text{Heatwave}, \text{Landslide}, \text{Soil Erosion}\}$
* $P(H_i | X_{\text{parcel}})$ is the calibrated probability of hazard $H_i$ occurring on the parcel within a 1-year window, generated by the spatial ML models.
* $w_i$ represents the hazard weight vector, adjusted dynamically by region (e.g., higher landslide weight in mountainous Himachal Pradesh; higher drought weight in arid Rajasthan). Default weights:
  $$\mathbf{w} = [w_{\text{flood}}=0.30, \, w_{\text{drought}}=0.25, \, w_{\text{heatwave}}=0.15, \, w_{\text{landslide}}=0.15, \, w_{\text{erosion}}=0.15]$$

### 1.2 Water Security Score (WSS)
The $WSS$ evaluates water availability and stress. It is formulated as:
$$WSS = 100 \times \left( \alpha_1 \cdot (1 - \text{GWS}) + \alpha_2 \cdot \text{WSI} + \alpha_3 \cdot \text{PPT}_{\text{anomaly}} + \alpha_4 \cdot \text{Prox}_{\text{water}} \right)$$
Where:
* $\text{GWS}$ is the Groundwater Stress index ($0\text{--}1$, from CGWB extraction ratios).
* $\text{WSI}$ is the Topographic Wetness Index scaled to $0\text{--}1$.
* $\text{PPT}_{\text{anomaly}}$ is the normalized precipitation anomaly (10-year rolling variance vs 100-year historical average).
* $\text{Prox}_{\text{water}}$ is the distance-decay factor to the nearest perennial reservoir/river:
  $$\text{Prox}_{\text{water}} = e^{-\lambda \cdot d(x)}, \quad \text{where } d(x) \text{ is distance in km, } \lambda = 0.2$$
* Scaling coefficients: $\alpha_1 = 0.40, \, \alpha_2 = 0.20, \, \alpha_3 = 0.20, \, \alpha_4 = 0.20$.

### 1.3 Land Development Suitability Score (LDSS)
The $LDSS$ measures engineering and construction viability, factoring in terrain constraints:
$$LDSS = 100 \times \left( \beta_1 \cdot f(\text{slope}) + \beta_2 \cdot (1 - P(\text{landslide})) + \beta_3 \cdot \text{Permeability} \right) \times (1 - I_{\text{flood\_zone}})$$
Where:
* $f(\text{slope})$ is a non-linear penalty function since flat terrain is highly suitable, while steep slopes are restrictive:
  $$f(\text{slope}) = \max\left(0, 1 - \left(\frac{\text{Slope (degrees)}}{20}\right)^2\right)$$
* $\text{Permeability}$ is soil infiltration speed ($0\text{--}1$).
* $I_{\text{flood\_zone}}$ is a binary indicator: $1$ if the parcel intersects a 10-year floodplain (based on SAR satellite backscatter history), and $0$ otherwise.

### 1.4 ESG Risk Score (ESG)
The ESG Score measures conservation value and environmental restrictions:
$$ESG = 100 \times \left( \gamma_1 \cdot \text{NDVI}_{\text{baseline}} + \gamma_2 \cdot \text{SOC} + \gamma_3 \cdot (1 - \text{LULC}_{\text{conversion}}) \right)$$
Where:
* $\text{NDVI}_{\text{baseline}}$ is baseline vegetation density.
* $\text{SOC}$ is Soil Organic Carbon content.
* $\text{LULC}_{\text{conversion}}$ is the biodiversity impact factor (e.g., high penalty if forest or wetland is converted to industrial; low penalty for barren/scrub land).

### 1.5 Investment Grade Score Map
Standardizing the numerical ratings into institutional-grade classifications:

| Grade | Risk Level | Score Criteria | Underwriting Action |
| :--- | :--- | :--- | :--- |
| **AAA** | Negligible | $CRS \le 15 \ \land \ WSS \ge 85 \ \land \ LDSS \ge 85 \ \land \ ESG \le 20$ | Auto-Approve, Standard terms |
| **AA** | Low | $CRS \le 25 \ \land \ WSS \ge 75 \ \land \ LDSS \ge 75 \ \land \ ESG \le 35$ | Auto-Approve, Standard terms |
| **A** | Stable | $CRS \le 35 \ \land \ WSS \ge 65 \ \land \ LDSS \ge 65 \ \land \ ESG \le 45$ | Approved, basic mitigation check |
| **BBB** | Moderate | $CRS \le 50 \ \land \ WSS \ge 50 \ \land \ LDSS \ge 50 \ \land \ ESG \le 60$ | Require detailed engineering check |
| **BB** | Vulnerable | $CRS > 50 \ \lor \ WSS < 50 \ \lor \ LDSS < 50 \ \lor \ ESG > 60$ | Underwriting Escalation / Rejected |

## 2. Spatial Machine Learning Models

Rather than running heavy physics-based hydraulic and geomorphological models for every lookup, we train fast **Spatial Machine Learning models** that approximate these physics runtimes.

```
+---------------------------------------------------------------------------------+
|                               Spatial ML Models                                 |
+----------------------+----------------------------------+-----------------------+
| XGBoost / LightGBM   | Graph Neural Networks (GNN)      | Bayesian Networks     |
| (For Tabular Stats)  | (For Hydro-Basin Catchment)      | (Causal Risk / UQ)    |
+-----------+----------+----------------+-----------------+-----------+-----------+
            |                           |                             |
            +---------------------------+-----------------------------+
                                        |
                                        v
                            [ Conformal Prediction ]
                                        |
                                        v
                     Probabilities & Calibrated 95% CIs
```

### Model 1: XGBoost / LightGBM (For Tabular Zonal Statistics)
* **Objective:** Predicts localized soil erosion probability, land suitability thresholds, and localized micro-heat risks.
* **Input Features:** Raster stats extracted via zonal query (Mean Copernicus DEM, Mean Slope, Aspect variance, Mean NDVI, Soil organic carbon, Permeability).
* **Target Label:** Historical erosional instances (classified from multi-temporal Sentinel-2 runs) and local temperature records exceeding thresholds ($T > 45^\circ\text{C}$).

### Model 2: Graph Neural Networks (GraphSAGE / GCN)
* **Objective:** Predicts Flood Risk and Groundwater Stress. Hydrology is directional and networked—a flood on a parcel depends on upstream topography and drainage networks.
* **Network Graph Formulation:** We build a directed graph $G = (V, E)$ where nodes $V$ represent land parcel centroids and edges $E$ represent hydrological flow paths (derived from HydroSHEDS DEM curvature).
* **Model:** We employ a GraphSAGE model to aggregate upstream environmental features (elevations, rainfall grids, drainage density) across spatial neighborhoods to predict downstream water accumulation:
  $$h_v^{(k)} = \sigma \left( \mathbf{W} \cdot \text{CONCAT}\left( h_v^{(k-1)}, \text{AGGREGATE}\left( \{h_u^{(k-1)}, \forall u \in \mathcal{N}(v)\} \right) \right) \right)$$
* **Advantage:** Captures watershed accumulation patterns that traditional non-spatial models (like random forests) miss.

### Model 3: Bayesian Networks (For Causal Risk & Uncertainty Quantification)
* **Objective:** Predicts Groundwater stress progression under CMIP6 climate warming projections.
* **Formulation:** Models the joint probability distribution of precipitation decline, agricultural pumping rates, soil water holding capacity, and policy restrictions. This allows us to query the network under counterfactual conditions: *What is the probability of critical water stress in 2050 if precipitation falls by 20% but irrigation efficiency rises by 10%?*

## 3. Confidence Intervals & Uncertainty Quantification

Climate risks are probabilistic; presenting a single score without error bounds is misleading. LND-11 uses **Conformal Prediction** to construct distribution-free confidence intervals.

For a predicted probability $\hat{y}$ (e.g., Flood probability of 0.24), we calculate a prediction set $\mathcal{C}(X)$ such that the true hazard outcome $Y$ is contained within the interval with a user-defined coverage probability $1-\epsilon$ (e.g., $95\%$ confidence):
$$P(Y \in \mathcal{C}(X)) \ge 1 - \epsilon$$

1. **Calibration Set:** We split our historical validation data into a training set and a calibration set ($N=10,000$ historical events).
2. **Non-conformity Score:** We define a non-conformity score $s_i$ for each validation point:
   $$s_i = |y_i - \hat{q}(x_i)|$$
   Where $\hat{q}(x_i)$ is the quantile regressed model output.
3. **Quantile Lookup:** We calculate the $(1-\epsilon)(1 + 1/N)$ quantile of the sorted scores $\{s_1, \dots, s_N\}$, denoted as $\hat{q}$.
4. **Interval Definition:** The confidence interval for any new parcel prediction is:
   $$\text{CI} = [\hat{y} - \hat{q}, \, \hat{y} + \hat{q}]$$

This guarantees that our risk intervals are mathematically sound and calibrated against true historical occurrences in India.

## 4. Model Training, MLflow & Triton Inference Pipeline

```
[ Git Repo / Data Lake ] ---> 1. Data Prep (Dask/Ray)
                                  |
                                  v
                             2. Model Training (PyTorch / LightGBM)
                                  |
                                  v
                             3. Registry & Versioning (MLflow)
                                  |
                                  v (Artifact Export: ONNX)
                             4. Server Deployment (Triton Server / SageMaker)
                                  |
                             5. Production API Requests (REST / gRPC)
```

1. **Data Pipeline (Ray/Dask):** Aggregates satellite features and spatial grids.
2. **Model Training:** PyTorch (for GNNs) and LightGBM models are trained on GPU clusters.
3. **MLflow Tracking:** Logs parameters (learning rate, depth), metrics (F1-score, Brier score), and serializes the model structure.
4. **ONNX Export:** Models are compiled to ONNX (Open Neural Network Exchange) format to optimize compute execution.
5. **Triton Inference Server:** Serves the ONNX runtime via gRPC to FastAPI workers, achieving inference latencies under $15\text{ms}$ per parcel lookup.
